ERP (唯一码克重/未上传检测网/加工费/一口价) 数据异常维护
100%
程序版本：W1 发布时间：2025.9.12
技术栈： RPA+ Excel + Python
执行频率： 定时自动执行
项目进度 100%
效益分析：
针对ERP中的唯一码、克重、未上传检测网、加工费、一口价这四项关键数据进行每日定时扫描、分类排查与智能预警，发现异常数据系统会自动生成预警报告，并通过钉钉机器人精准发送至对应的数据负责人。报告明确列出异常数量、异常类型，实现“一事一报，责任到人”。
每个排查模块都能在准确的数据子集上运行，原本需要人工耗时数小时的数据核对，实现至系统自动完成，效率提升超过99%。
【唯一码克重产品异常维护】每日早上11点-从颜色规格提取产品重量区间，修正 ±0.03g(足金/18K/铂金和银饰修正逻辑相同，只是容差从 ±0.03g 改成 ±1g)，异常数据判定条件：产品重量没有在修正区间并且有仓位数据的。
【二维码未上传异常产品维护】每日早上9点-异常数据判定条件：二次编码为空或0并且有仓位数据的。
【足金、K金、铂金无工费数据维护】每日早上10点-异常数据判定条件：加工费单价，附加费为空或0并且有仓位数据的，三个条件同时满足为异常数据。
【银饰、铂金、玉石、手表单价维护】每日早上10点-异常数据判定条件：单价为空或0并且有仓位数据的。

💰 黄金定价规则

金镶玉 / 锆：

  • 重量 ≤ 0.5 → 3988
  • 0.5 < 重量 ≤ 1.0 → 3288
  • 1.0 < 重量 ≤ 2.0 → 2988
  • 重量 > 2.0 → 2588

足金编绳款（颈绳 / 手绳 / 串珠）：

  • 重量 ≤ 0.5 → 2500
  • 0.5 < 重量 ≤ 1.0 → 2200
  • 1.0 < 重量 ≤ 2.0 → 2000
  • 重量 > 2.0 → 1800

足金素金：

  • 重量 ≤ 0.5 → 一口价 2200
  • 0.5 < 重量 ≤ 1.0 → 一口价 2000
  • 1.0 < 重量 ≤ 2.0 → 1800
  • 重量 > 2.0 → 1600

💎公式原则

  • 可用库存 = 实际可用数 - 售后仓-维修仓 + 30天内采购在途 + 15天内调拨中货品
  • 安全库存计算 = 7日日均销量 × 工厂工期
  • 建议补货量 = 安全库存 - 可用库存
